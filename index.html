URIs: Idea

  • URI = Uniform Resource Identifier
  • Serves as a global unique identifier of resource
  • Any object which has a clear identity can be a resource (e.g. books, places, people, abstract concepts, etc.)
  • Similar thing already exists in certain domains: ISBN for books

URIs: Syntax

  • Extension of the URL concept.
    But not every URI denotes a Web document (conversely, the URI of Web documents often used as their URL).
  • Begins with the the URI scheme separated by ":" from the subsequent part (eg: http, ftp, mailto)
  • Any object which has a clear identity can be a resource (e.g. books, places, people, abstract concepts, etc.)
  • Every URI consists of four parts, as follows:
    Scheme : Hierarchy [ ? Query ] [ # Fragment ]

URIs: Self-Describing

  • Necessary when a resource has no URI (or URI is not known).
  • Strategy to avoid unwanted overlap: use of http-URIs for the website.
  • Allows dropping a URI documentation.
  • Example: http://jens-lehmann.org/foaf.rdf#me

IRIs

  • IRI = Internationalized Resource Identifier
  • Represent a generalization of the URI
  • IRIs can contain Unicode characters
  • Examples:
    http://www.example.org/Wüste
    http://www.example.org/사막
    http://пример.ру/Санкт-Петербург

Other Identifier Systems

  • URN = Uniform Resource Name
  • Subset of URIs
  • Used to identify resources by name
  • Used for globally unique and persistent identifier of goods (e.g. ISBN)
  • Example:
    URN of "Science of Computer Programming" journal - urn:issn:0167-6423

Other Identifier Systems

  • ISBN = International Standard Book Number
    ISSN = International Standard Serial Number
  • ISBN used to identify books
    ISSN is used to identify journals and book series
  • Example:
    ISBN 978-3-86680-192-9
    ISSN 1234-5678

Other Identifier Systems

  • DOI = Digital Object Identifier
  • DOI is a unique and permanent identifier for digital objects.
    Mainly used for on-line articles and scientific journals.
  • Example:
    DOI Handbook - DOI 10.1000/182

General information on RDF

Resource Description Framework

W3C Recommendation (http://www.w3.org/RDF)

RDF is a data model

  • Originally was made to provide metadata for Web resources
  • Coded structured information
  • Universal, machine-readable data format

Components of RDF graph

  • URI

    for unique referencing to resources

  • Literals

    for representing an instance data value

  • Blank nodes

    for representing a resource for which a URI or literal is not given

Example of RDF graph

Literals

Used for the representation of data values

View as strings

Graph as a set of triples

Graph as a set of triples

RDF Triple

Components of an RDF triple

Modeled on linguistic categories, but not always consistent

Allowed assignments:

  • Subject: URI or Blank node
  • Predicate: URI (also called Property)
  • Object: URI or Blank node or Literal

Node and edge identifiers are unique, so the original graph can be always reconstructed from triple-list

Syntax: Turtle

Turtle syntax:

  • URIs written in angle brackets, for example: <http://dbpedia.org/resource/Leipzig>
  • Literals written in quotes, for example: "Leipzig"@de and "51.333332"^^xsd:float
  • Triples are completed by dot
  • Spaces and line breaks within identifiers are ignored

Schema knowledge with RDFS

RDF Schema (RDFS):

  • Vocabulary is not thematized (as in FOAF), but generically (in terms of re-use)
  • Allows the specification of (parts of) the semantics of arbitrary RDF-vocabularies (that is a kind of "Metavokabular")
  • Advantage is that the software with RDFS support interprets each defined by means of RDFS vocabulary correctly
  • Functionality makes RDFS into an ontology (lightweight ontology)
  • „A little semantics goes a long way.“
